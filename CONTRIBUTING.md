# Contributing to TradePilot OS

Thank you for your interest in contributing to TradePilot OS.

## Contribution Principles

- Follow the project coding standards.
- Follow the documentation standards.
- Every change must be traceable to a task or issue.
- Do not commit generated files unless required.
- Keep commits focused and descriptive.

## Development Workflow

1. Pull the latest changes from the repository.
2. Create or update the relevant implementation task.
3. Complete the implementation.
4. Validate the solution.
5. Commit using the approved commit message format.
6. Push to GitHub.

## Commit Message Format

We follow Conventional Commits.

Examples:

```text
feat(authentication): implement login window
fix(database): resolve SQLite connection issue
docs(strategy): update product vision
chore(project): update dependencies
```

## Code Quality

All code should:

- Be readable
- Include type hints where appropriate
- Avoid duplication
- Follow project architecture
- Pass validation before commit

## Documentation

Documentation is part of the implementation.

Every significant architectural or design change must be documented.

Thank you for helping improve TradePilot OS.
